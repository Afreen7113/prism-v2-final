## DEFAULT THEME & GLOBAL HOVER LEAK AUDIT

### PROBLEM 1: Root Cause Investigation

1. **Which theme is active on initial page load:** The hardcoded `defaultTheme` state object.
2. **Where the default theme is being assigned:** During SSR and initial state initialization in `providers/ThemeProvider.tsx`.
3. **Whether localStorage, ThemeProvider, cookies, or fallback logic are forcing a hidden/default preset:** `ThemeProvider` forces the initial React state to `defaultTheme` before `useIsomorphicLayoutEffect` has a chance to read `localStorage` or system preferences on the client.
4. **Which preset is responsible for the purple appearance:** The `defaultTheme` object itself is hardcoded with `var(--color-indigo-500)` as the `primaryColor`, causing the purple/indigo leak before the active preset is applied.
5. **Exact file and line number:** `providers/ThemeProvider.tsx`, lines 33-54 (definition) and line 207 (initial state assignment).
